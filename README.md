# 下課狀態看板

這是教室下課時間用的全螢幕看板。老師可以看見每位學生是否可以下課，也可以同步 Google 試算表中的未交作業狀態。

## 重要提醒

不要直接雙擊 `index.html` 開啟。請用 `localhost` 或 VSCode Live Server。

直接用 `file://` 開啟時，瀏覽器可能會擋 Google Sheet 或 Apps Script，同步就容易失敗。

## 用 npm start 開啟

1. 打開終端機，切到本資料夾。
2. 執行：

```bash
npm start
```

3. 瀏覽器開啟：

```text
http://localhost:4173
```

## 用 VSCode Live Server 開啟

1. 用 VSCode 開啟本資料夾。
2. 安裝 Live Server 擴充套件。
3. 在 `index.html` 上按右鍵。
4. 選 `Open with Live Server`。

## Google Sheet 格式

每個頁籤都用同一種格式：

- A 欄：座號
- B 欄：姓名
- C 欄之後：作業繳交狀況
- 第 1 列：作業名稱
- 第 2 列開始：學生資料

同步時只會加入「A 欄是數字座號」且「B 欄有姓名」的資料列。作業標題、空白列、說明文字不會被當成學生。

如果某位學生在 C 欄之後有紅色格子，看板會判定為「下課冷靜」，並在「作業」按鈕中顯示該欄第 1 列的作業名稱。

## 紅色格子設定

建議使用這些紅色：

- `#ff0000`
- `#ea4335`
- Google Sheets 中接近純紅、不是粉紅也不是橘色的紅色

程式會避免把粉紅色、橘色誤判成未交。

## Apps Script 部署

1. 打開 Google 試算表。
2. 選「擴充功能」→「Apps Script」。
3. 貼上本資料夾的 `apps-script.gs` 內容。
4. 按「部署」→「新增部署作業」。
5. 類型選「網頁應用程式」。
6. 執行身分選「我」。
7. 存取權限選「知道連結的任何人」。
8. 複製網頁應用程式網址。
9. 回到看板右上角齒輪，貼到「Apps Script 網頁應用程式網址」。

## Google Sheet 權限

如果只貼 Google Sheet 網址，試算表需要設定為「知道連結的任何人可查看」。但這種方式讀不到格子顏色，只能讀文字。

要讀紅色格子，請使用 Apps Script。

## 常見錯誤

- 看到「請不要直接雙擊 index.html」：改用 `npm start` 或 Live Server。
- 看到「Apps Script 網址看起來不正確」：請貼部署後的網頁應用程式網址，不是 Apps Script 編輯器網址。
- 看到「Apps Script 回傳的不是 JSON」：通常是貼錯網址、部署權限不對，或程式碼沒有貼完整。
- 看到「CORS 被擋」：請確認用 localhost 開啟，並重新部署 Apps Script。
- 看到「沒有讀到學生資料」：請檢查 A 欄座號、B 欄姓名、第 2 列開始是否有學生。

## 使用方式

- 點學生卡片：手動切換「可以下課 / 下課冷靜」。
- 點「作業」：查看該學生未交作業。
- 點主畫面「全班冷靜」：主畫面會顯示大型「全班冷靜」提示，不會改掉每位學生原本狀態；再按「取消冷靜」即可回復原本狀態。
- 點主畫面「全螢幕」：進入全螢幕；如果老師設定面板開著，會先自動關閉。
- 右上角齒輪：設定學生人數、同步網址、分時段提醒、每週固定叮嚀與今日臨時提醒。
